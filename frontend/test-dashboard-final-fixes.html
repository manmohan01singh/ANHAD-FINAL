📊 Fix 1: Target Line Position FIXED

✅ Issue: Target line was at bottom (20% from top)

The target line was positioned at top: 20% which placed it near the bottom of the chart. Users expected it at the TOP where bars should reach UP to.

✅ Solution: Dynamic calculation based on goal percentages

Now calculates the average target percentage across all three metrics (Read, Listen, Nitnem) and positions the line at approximately 70-80% height (20-30% from top in CSS).

// Calculate target line position (inverted) const avgTargetPercent = ((TARGET_PAGES / maxRead) + (TARGET_MINUTES / maxListen) + (TARGET_NITNEM / maxNitnem)) / 3; const targetLinePosition = 100 - (avgTargetPercent * 100); const targetLineTop = Math.max(15, Math.min(35, targetLinePosition));

✅ Expected Result

Target line appears at the upper portion of the chart (15-35% from top). Bars grow upward toward this line. "GOAL" label is visible on the right side.

📿 Fix 3: Nitnem Display Format FIXED

✅ Issue: Nitnem showing "0/1" instead of "X/Y banis"

The Nitnem goal was displaying as "0/1" which didn't reflect the actual number of banis selected and completed.

✅ Solution: Show actual bani counts

Now reads from Nitnem Tracker's selected banis and completed log to show the real progress like "5/9 banis" or "3/5 banis".

// Get Nitnem progress for better display const nitnemLog = JSON.parse(localStorage.getItem('nitnemTracker_nitnemLog') || '{}'); const selectedBanis = JSON.parse(localStorage.getItem('nitnemTracker_selectedBanis') || '{"amritvela":[],"rehras":[],"sohila":[]}'); const totalBanis = (selectedBanis.amritvela?.length || 0) + (selectedBanis.rehras?.length || 0) + (selectedBanis.sohila?.length || 0); // Show X/Y banis format if (key === 'completeNitnem' && totalBanis > 0) { statusText = `${completedBanis}/${totalBanis} banis`; }

✅ Expected Result

Nitnem goal card shows "X/Y banis" format (e.g., "5/9 banis"). Progress bar reflects actual completion percentage.

✅ Issue: Nitnem bar going off-chart

When Nitnem count was high (e.g., 9 banis), the bar would exceed the chart height because the scale wasn't accounting for the target line properly.

✅ Solution: Better max value calculation

Uses the maximum of actual data OR target goals for scaling, with a minimum scale of 10 for Nitnem. Bars are capped at 100% height.

// Use targets as minimum scale, or actual max if higher const maxRead = Math.max(...data.map(d => d.readPages), TARGET_PAGES); const maxListen = Math.max(...data.map(d => d.listenMinutes), TARGET_MINUTES); const maxNitnem = Math.max(...data.map(d => d.nitnemCount), TARGET_NITNEM, 10); // Cap bars at 100% style="height: ${Math.min((day.readPages / maxRead) * 100, 100)}%"

✅ Expected Result

All bars stay within chart boundaries. No overflow or clipping. Bars scale proportionally to show relative progress.
